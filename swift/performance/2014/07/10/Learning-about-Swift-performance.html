Learning about Swift performance

10 Jul 2014

Blog post over at http://www.swift-studies.com looking at the process of refactoring OysterKit in order to improve its performance. Nice tip about XCode6's measureBlock(){} feature. The main findings of the investigation were

  • Profile your code to work out performance issues (don't just assume that it is Swift's fault, it probably isn't)
  • String interpolation in Swift is slow at this stage (not surprising)
